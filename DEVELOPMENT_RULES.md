# 開發規範與協作規則 (Development Rules & Guidelines)

此檔案紀錄了本專案開發過程中的核心規範，AI 助手在每次啟動或執行任務前必須主動讀取並嚴格遵守。

## 1. 程式碼撰寫與上傳規則
- **核心指令**：除非使用者明確輸入「**開始**」二字，否則 AI 助手不得自行撰寫、修改或刪除任何專案程式碼。
- **自動上傳**：在完成階段性開發或使用者要求的改動後，應自動執行 Git Commit 並 Push 到 GitHub 遠端倉庫，以確保版本即時同步。
- **非授權期間行為**：在尚未獲得「開始」指令前，AI 僅能進行分析、討論或規劃。

## 2. 專案背景與技術棧
- **專案名稱**：暗棋 (Dark Chess)
- **技術棧**：原生 HTML / CSS / JavaScript (Vanilla JS)。
- **風格要求**：高質感的現代網頁設計，包含動態效果與細膩的視覺美感。

## 3. 持續更新
- 當有新的協作規範或開發偏好時，應同步更新於此檔案中。
